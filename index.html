A VARIABLE IS LIKE A LABELED BOX!
You give the box a name (the label) and put a value (stuff) inside it. You can change what's inside anytime! That's why it's a variable — it can vary!
// Create a variable and put something in it let playerName = "Cody" // Text (a string) let bananas = 5 // A number let isHungry = true // True or false (boolean) // Change what's inside! bananas = 10 // Now Cody has 10 bananas! 🍌
📝

"Hello!"
🔢
NUMBER
Any number
42 or 3.14
✅
BOOLEAN
True or False
true / false
KEY CONCEPT: A VARIABLE is a named container that stores a value. The value can change over time!

CHAPTER 3
LOOPS — THE VINE SWING! 🔁
Cody needs to cross a canyon by swinging on 5 vines...
AT THE CANYON'S EDGE...
I need to swing across 5 vines! That's a LOT of writing! 😰
🌿🐒 🌿 🌿 🌿 🌿
Without a loop I'd write "swing" five times... wasteful!
swing() swing() swing() swing() swing() // Yikes! 😫
USE A LOOP!
🦜
Instead of writing "swing" 5 times, tell the computer: "REPEAT swing 5 TIMES!"
for (let vine = 1; vine <= 5; vine++) { swing() // This runs 5 times! ✨ }

There are also WHILE loops — they keep going UNTIL something becomes false!
while (hungry == true) { eatBanana() // Keep eating! }
KEY CONCEPT: A LOOP repeats instructions multiple times. FOR loops count, WHILE loops check a condition!

Count carefully: i starts at 1 and goes up to (and including) 4. That's 4 rounds!

CONDITIONALS LET YOU MAKE DECISIONS!
An IF statement checks if something is true or false, then does different things. It's like "choose your own adventure" for code!
THE CODE:

A FUNCTION IS A REUSABLE BLOCK OF CODE!
Write the code once, give it a name, then call it whenever you need it! Functions can take inputs (parameters) and give back outputs!
SPELL 1: COLLECT BANANAS
// Define the spell function collectBananas() { climbTree() grabBananas() climbDown() } // Cast it anytime! collectBananas() // 🍌 collectBananas() // 🍌🍌
SPELL 2: WITH INPUTS!
// This spell takes a number! function jump(times) { for (let i = 0; i < times; i++) { leapUp() } } jump(3) // Jumps 3 times jump(10) // Jumps 10 times!

WHAT IS A BUG?
A bug is a mistake in code that makes it do the wrong thing. Debugging means finding and fixing those mistakes. Every coder debugs daily!
🔤
SYNTAX BUG
Spelling or typing error
// Wrong! functoin hello() {}
🧠
LOGIC BUG
Code runs but does wrong thing
// Meant to add! score = score - 10
🔢
ORDER BUG
Steps in wrong order
// Can't eat first! eat() cook()
KEY CONCEPT: DEBUGGING is finding and fixing mistakes. Read → Find → Fix! It's a coder's superpower! 🦸

📖 JUNGLE GLOSSARY

Click any term to reveal its definition!

ALGORITHM
A set of step-by-step instructions to solve a problem. Like a recipe for code!
BOOLEAN
A data type that can only be true or false. Like a light switch!
BUG
A mistake in code that causes incorrect behavior. Named after a real moth in an early computer!
CODE / CODING
Instructions written in a programming language that tell a computer what to do.
CONDITIONAL
A statement (IF/ELSE) that runs different code depending on whether a condition is true or false.
DATA TYPE
The kind of value stored in a variable: string (text), number, or boolean.
DEBUGGING
The process of finding and fixing bugs in code. Read → Find → Fix!
FOR LOOP
A loop that repeats code a specific number of times. You set the start, end, and step.
FUNCTION
A named, reusable block of code. Define once, use many times! Can take inputs (parameters).
IF STATEMENT
Checks if a condition is true, then runs code inside. "If this is true, do that."
INFINITE LOOP
A loop that never stops because the condition is always true. Usually a bug!
LOOP
A structure that repeats a block of code multiple times. FOR loops count, WHILE loops check conditions.
NUMBER
A data type for numerical values like 42 or 3.14. Used for math and counting.
PARAMETER
A value passed into a function, giving it information to work with.
STRING
A data type for text, always wrapped in quotes: "Hello!"
VARIABLE
A named container that stores a value. The value can change (vary) over time.
WHILE LOOP
A loop that repeats as long as a condition remains true.
SYNTAX
The rules for how code must be written. Like grammar for programming languages.
